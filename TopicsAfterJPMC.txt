SOLID Desing Principles:
	Single Responsibility: Every class should have single responcibility
	Open & Close: According to new requirement the module should be Open for extension but close for Close for Modificatoin
					Every class should be Open for extension but Close for Modification 
	Leskov's Substitution Principle: It applies to inheritance in such away that Drived/Sub class can be substitutable with its Base/Parent class without any issue.
	Interface Segregation Principle: We should not force client/class to implementat all the method if they don't want to use it instead larger interfaces should be splitted into 
										smaller onces. This is similar to Single Responsibility Principle
	Dependency Inversion Principle: We must use abstraction(abstract class or interface) instead of concrete implementation, High level module should not be dependent on low 
									level module but both should dependent on on Abstraction.
